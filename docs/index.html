Healthcodec é um formato compacto e eficiente para armazenar dado em saúde registrado em conformidade com objetos do modelo de referência do openEHR. O projeto também contempla os tipos de dados empregados pelo FHIR.

Google faz uso de Protocol Buffers e FlatBuffers, Netflix usa Hollow, Github usa BERT, Apache usa Avro, Redis usa MessagePack, Kryo serializa grafos de objetos em Java, e ORC dentre outras opções.

Healthcodec é uma estratégia específica para dados em saúde, e mais eficiente.
